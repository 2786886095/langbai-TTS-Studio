# langbai-TTS-Studio 可执行验收计划

基线日期：2026-07-19
目标平台：Windows 桌面端（Electron + React + Python）

## 1. 验收结论规则

验收分为四道门，前一道不通过时不能宣称后一阶段完成。

1. **G0 静态契约**：项目可安装、配置与参数目录完整，测试可在 Mock 模式运行。
2. **G1 Mock 业务闭环**：不加载真实模型，完成创建任务、分段、重试、断点恢复、合并、取消与错误展示。
3. **G2 真实引擎闭环**：IndexTTS2、VoxCPM、GPT-SoVITS 分别至少生成一个可播放 WAV，参数确实传入对应引擎。
4. **G3 Windows 成品**：打包后的 Windows 应用可启动，真实完成短音频与长音频任务，UI 和恢复行为通过人工验收。

任何只通过 G0/G1 的构建都不能表述为“三引擎可用”。真实模型缺失、环境不可启动或只展示参数但未透传，均判定 G2 失败。

## 2. P0 功能清单

### 2.1 桌面应用与任务

- [ ] Windows 10/11 x64 上双击启动，不要求用户手工开启前端或 Python 服务。
- [ ] Electron 仅绑定回环地址；端口冲突时能自动处理并给出中文错误。
- [ ] 一个任务必须且只能选择 `indextts2`、`voxcpm`、`gpt_sovits` 之一。
- [ ] 任务状态至少包括 `queued`、`running`、`completed`、`failed`、`cancelled`，取消不归类为失败。
- [ ] 同一 GPU 默认串行执行；不会同时常驻三套大模型导致显存无界增长。
- [ ] 关闭窗口或后端异常后，已落盘任务不会丢失。

### 2.2 参数完整性与中文说明

- [ ] 引擎参数表至少覆盖 `tests/acceptance/engine_parameter_baseline.json` 的所有 `required_surface` 参数。
- [ ] 每个参数均有非空中文名称、用途说明；枚举/布尔参数说明各选项效果。
- [ ] 参数默认值、范围、互斥关系与当前本地官方入口一致。
- [ ] 隐藏或禁用不适用于当前模型版本的参数，并说明原因，不能静默丢弃。
- [ ] 提交任务后的 manifest 保存解析后的完整参数快照；随后修改 UI 不影响旧任务。
- [ ] 后端适配器收到的参数与任务快照一致；未知参数返回明确的 4xx，不静默忽略。

### 2.3 长音频

- [ ] 按标点优先、最大长度兜底分段；分段后按顺序拼接可还原规范化正文。
- [ ] 中文标点、英文标点、换行、超长无标点文本、空白文本与 emoji 均有测试。
- [ ] 每段具有稳定 ID、序号、文本、状态、尝试次数、错误与输出路径。
- [ ] 单段失败按配置自动重试，不重新生成已经成功的段。
- [ ] 应用/后端重启后从 manifest 恢复，仅续作未完成或失败段。
- [ ] 最终合并前统一采样率、位深与声道；顺序正确，无截断，段间静音符合设置。
- [ ] 生成最终 WAV 的同时保留全部成功分段；合并采用临时文件加原子替换，失败时不留下伪完成文件。
- [ ] 完成条件为：所有段成功、最终音频可解码、时长大于零、manifest 已落盘。

### 2.4 结果与错误

- [ ] 列表与详情实时显示进度（完成段数/总段数）和当前阶段。
- [ ] 支持试听最终音频、打开输出目录、查看保留的分段。
- [ ] 错误信息包含引擎、段序号、可执行原因与日志入口；不得只显示 `生成失败`。
- [ ] 手动重试失败任务不覆盖旧诊断信息，尝试次数递增。
- [ ] 路径包含中文和空格时，参考音频、任务目录和最终输出均正常。

## 3. UI 人工验收（G3）

- [ ] 纯色视觉，不使用渐变；颜色、字重、间距、圆角与阴影有一致 token。
- [ ] 1280x720、1440x900、1920x1080 下无横向溢出、遮挡或不可达控件。
- [ ] 核心路径（选引擎 → 填文本/参考音频 → 调参数 → 生成 → 试听）有清晰主次。
- [ ] 完整参数可检索/分组；常用参数与高级参数层级清楚，但高级区不是缺参借口。
- [ ] 每个参数的中文用途可在不离开当前表单的情况下查看。
- [ ] 键盘可遍历交互控件；焦点清晰；文本与背景对比度满足 WCAG AA 的常规文本要求。
- [ ] 加载、空状态、失败、取消、恢复和超长文本状态均有实际截图证据。
- [ ] 通过 Electron 实际窗口截图验收，不能只看源码或浏览器静态页面。

## 4. 自动化测试分层

在项目根目录执行：

```powershell
python -m pytest tests/acceptance -q
python -m pytest tests/acceptance -q --require-implementation
python -m pytest tests/acceptance -q --require-implementation --live-engines
```

- 默认：校验验收基线自身，尚未出现的实现测试会跳过。
- `--require-implementation`：缺少后端、参数目录或任务 API 时直接失败，用于 G0/G1。
- `--live-engines`：开启真实模型短音频测试，用于 G2；不得在普通 CI 意外下载模型或占用 GPU。

最终还需运行项目自身的前端检查、Python 单元测试、Electron 打包及 Windows 安装包烟测。具体命令以项目落地后的 README/package scripts 为准。

## 6. 已识别的验收风险

1. **依赖冲突**：三项目的 PyTorch、Transformers、音频库与 CUDA 扩展可能不兼容。推荐每引擎独立 Python 环境/子进程，验收时检查真实解释器路径。
2. **显存峰值**：同时加载三模型不等于“可同时调用”。产品需求是任务择一，应串行调度并支持卸载/切换。
3. **参数表面完整但未透传**：UI 参数必须以 adapter spy 或 manifest + 调用日志证明传入，不以控件存在作为通过。
4. **双重分段**：Studio 长文本分段与 IndexTTS/GPT-SoVITS 自带分段可能叠加。必须规定外层每段调用的内部分段参数，并验证顺序和边界。
5. **采样率差异**：不同引擎/版本输出采样率不同；简单拼接字节会损坏 WAV，必须解码、重采样后重编码。
6. **GPT-SoVITS 版本参数差异**：`sample_steps`、`super_sampling`、语言列表等依模型版本变化，UI 应基于能力动态启用。
7. **参考音频语义不同**：VoxCPM 的 `prompt_wav_path + prompt_text` 与 `reference_wav_path` 语义不同；IndexTTS 的说话人参考与情感参考也不同，不能合并为一个模糊字段。
8. **Windows 路径与进程清理**：中文路径、空格、反斜杠、长路径、子进程残留、端口残留都需在打包应用中实测。
9. **断点恢复一致性**：只写数据库/JSON 但未原子落盘可能在断电时损坏；恢复验收应模拟进程在段完成前后被终止。
10. **取消语义**：GPU 推理通常无法瞬时中断；应表现为“正在停止/已取消”，不得让取消后的迟到结果把任务改回完成。

## 7. 最终证据包

最终验收报告至少包含：

- 安装包路径、版本、Git 提交与构建命令；
- 自动化测试原始输出；
- 三引擎各一份真实 WAV 的路径、采样率、时长、文件哈希与使用参数；
- 一份长音频任务的 manifest、全部分段和最终合并 WAV；
- 一次失败重试、一次重启续作、一次取消的日志/manifest 证据；
- 关键分辨率和各状态的 Electron 窗口截图；
- 已知限制，不得把跳过项写成通过项。
