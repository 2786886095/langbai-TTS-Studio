# 商业级完成度基线审计

审计日期：2026-07-19
审计对象：Windows 打包版 `langbai-TTS-Studio 1.0.0`
目标环境：1920×1080、Windows 100% 缩放；正文默认不低于 14px，主要操作 15–16px，并提供紧凑模式。

## 审计证据

### 01 创作台

![创作台基线](audit/commercial-baseline/01-studio.png)

总体状态：核心生成入口可见，但仍是高密度技术面板，未达到舒适、可信的日常生产工具标准。

- 优点：引擎、文本、参数、队列处于同一工作上下文；主要生成按钮位置明确。
- UX 风险：任务队列、音频库、历史记录虽在主导航中出现，当前实现没有独立产品页面；保存方案、导入 TXT、粘贴纯文本等可见入口缺少完整反馈闭环。
- 可读性风险：参数帮助、状态、统计和队列空状态大量使用约 8–10px 字体；右侧参数面板过窄且滚动负担高。
- 无障碍风险：小字号、小点击目标、淡灰辅助文字和仅靠颜色表达的状态可能影响低视力、缩放和键盘用户。截图不能证明完整 WCAG 合规，仍需 DOM、键盘与高 DPI 测试。
- 信任风险：启动阶段三个引擎同时显示“检测中”，但没有清晰的阶段、耗时预期或诊断入口。

### 02 引擎管理

![引擎管理基线](audit/commercial-baseline/02-engine-manager.png)

总体状态：安装结构方向正确，但字号、加载状态、信息架构和错误恢复仍偏工程内部工具。

- 优点：源码、Python 环境、模型三层状态分离；许可证确认与模型独立下载被明确呈现。
- UX 风险：页面在接口返回前展示后备模型和“等待清单”文案，容易让用户把临时占位误认为真实官方清单。
- 可读性风险：表头、许可证、路径、来源、状态等关键内容过小；页面出现全局滚动，重点安装操作与诊断日志可能离开视野。
- 操作风险：设备选择缺少显卡检测和推荐说明；缺少安装前容量汇总、依赖工具状态、失败后的可执行修复建议。
- 无障碍风险：复选框和表格操作目标偏小；状态标签主要依赖颜色；焦点顺序和键盘操作尚无证据。

## 必须完成的产品闭环

| 范围 | 完成证据 |
| --- | --- |
| 字体与密度 | 舒适模式正文≥14px、主要操作15–16px；紧凑模式可切换；截图与静态规则共同验证 |
| 导航 | 创作台、任务、音频库、历史、设置均有真实独立内容和空/加载/错误状态 |
| 项目 | 新建、保存、自动保存、恢复、复制、重命名、删除均持久化且重启后存在 |
| 音频 | 完成任务可播放、暂停、定位、显示时长/格式、打开目录和导出 |
| 任务 | 排队、运行、成功、失败、取消、重试和恢复状态真实可见 |
| 引擎安装 | 预检、许可证、容量、依赖、进度、取消/继续、错误建议、完整性校验形成闭环 |
| 设置 | 密度、输出目录、默认引擎、更新策略、日志与诊断持久化 |
| 首次启动 | 真实引擎检测、安装/连接选择、模型说明和完成页 |
| 桌面体验 | 高 DPI、键盘快捷键、窗口状态恢复、外链安全、退出后无残留进程 |
| 发布 | 自动化构建、版本信息、更新契约、签名状态与许可证说明清楚 |

## 当前证据限制

基线截图只能确认可见层级、密度和部分状态，不能单独证明键盘访问、读屏语义、持久化、安装恢复或真实生成。商业级结论必须同时依赖运行后的逐屏截图、API/持久化测试、键盘与缩放检查，以及最终打包版的完整流程测试。
