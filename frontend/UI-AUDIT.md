# 商业级 UI 重构审计

审计日期：2026-07-19

## 审计限制

本子任务没有可控浏览器截图能力，因此重构前审计证据来自 `src/styles.css` 和现有 React 交互路径，不将其描述为截图审计。最终视觉结论仍需使用 Electron 实际窗口截图确认。

## 重构前主要问题

1. 大量正文、帮助信息和状态文字使用 7–11px 字号，参数说明、安装状态和任务队列不适合长时间桌面使用。
2. 多个按钮、开关、复选框和图标按钮只有 19–34px，低于舒适点击目标。
3. “任务队列”“音频库”“历史记录”只改变侧栏选中态，没有真实页面和数据状态。
4. 缺少首次启动引导，以及通用的加载、错误、空状态和危险操作确认。
5. 只有固定高密度布局，用户无法选择舒适或紧凑密度。

## 已实施

- 舒适模式：正文与表单标签不低于 14px，主要操作 15px；交互目标统一不低于 40px。
- 紧凑模式：保留桌面高密度效率，所有辅助文字仍不低于 12px，交互目标仍不低于 40px；密度选择持久化。
- 任务队列读取 `/api/jobs`，音频库读取 `/api/library/audio`，历史记录读取 `/api/history`，具备加载、错误、搜索、空状态和任务操作。
- 取消任务前显示确认对话框；失败和已取消任务支持真实重试。
- 播放和定位音频前读取 `/api/jobs/{id}/output`，再交给 Electron 的受控播放/定位方法。
- 保存方案通过 `/api/projects` 创建或更新真实项目，不再伪造本地保存成功。
- 首次启动展示三步引导，可直接进入引擎管理。
- 设置/引擎管理沿用真实安装、下载、许可证和错误日志接口，并接入带 revision 冲突处理的 `/api/settings`、`/api/diagnostics/exports` 与 Electron 更新事件。
- 移除模型占位清单；后端未返回模型时明确显示空状态。

## 可截图入口

- 启动入口：运行桌面端后默认进入“创作台”；清除 `localStorage.langbai-onboarding-complete` 后重启可进入首次引导。
- 侧栏入口：“创作台”“任务队列”“音频库”“历史记录”“设置与路径”。
- 设置状态：“设置与路径”顶部为桌面偏好、持久化工作区设置、诊断导出；下方为引擎安装与下载。
- 确认态：“任务队列”中对运行任务点击取消按钮。
- 音频播放态：“音频库”点击任一可用音频的播放按钮。
- 错误/空状态：停止本地后端后打开任一数据页；恢复后点击“重试”。
